# チケット仕様確認

<Info>決定事項は他の人が読んでもわかるようにBacklogにまとめましょう。</Info>

## コミュケーション

チケットの仕様を確認するために、コミュニケーションが発生します。
Figma、Backlog、GitHub でコメントした場合は チャット でもメンション飛ばすようにしましょう。
ツール上のメンションでは気が付かないことがあるので、チャットでもメンションして伝えるようにしましょう。

どのツールでやりとりするかは、ケースバースケースで決めて OK です。
スピーディーにストレスなく相手と適切なコミュニケーションが取れる手段を取れば OK です。

## 確定した要件は

議論の上、確認した要件は、必ずバックログにまとめるようにします。
更に、要件だけではなく、その要件に決まった経緯が知りたいこともあるので、
チャットで議論を行った場合、チケットにリンクを貼ってトレース可能にしておきましょう。
口頭で議論を行った場合も、必ずなせその仕様にしたのか記録を残すようにしましょう。
