# 期待値理解

開発者が正しく仕様を理解し、それに基づいて開発を行うことを指します。

## 期待値理解をするには
弊社では、プロジェクトの仕様について最も詳しく精通しているのはデザイナーです。
そのため、期待値理解を高めるためには、デザイナーとのコミュニケーションを重視する必要があります。
仕様が不明確な場合は、直ちにデザイナーに確認するようにしましょう。

## なぜ期待値理解が必要なのか
期待値理解が不足していると、プロジェクトの成功や品質向上に悪影響を及ぼす可能性があります。具体的には以下のような問題が発生します。
- 誤った実装を行い、修正に多くの時間やリソースが増えてしまう。
- レビュアーが実装内容を理解するための負担が増えてしまう。
- バグの発生や品質の低下が生じる可能性がある。
- プロジェクトの納期が遅れる可能性がある。

## 未然に問題を防ぐには
未然に問題を防ぐには、プルリクエストにデザイナーを積極的に参加させることが重要です。これにより以下のような問題を未然に防ぐことができます。
- 開発者が正しい仕様を理解して実装する確率が高まる。
- デザイナーが早い段階で実装内容をチェックできるため、修正が早期に行われる可能性がある。
- レビュアーの負担が軽減され、よりスムーズなレビュープロセスが実現できる。
- バグや品質の問題が発生しにくくなる。
- プロジェクトの進行がスムーズになり、納期遅延のリスクを減らすことができる。

期待値理解の向上は、プロジェクトの成功や品質向上に直結する重要な要素です。デザイナーとの綿密なコミュニケーションを通じて、正確な仕様理解を図ることを心掛けましょう。
