# なぜ工数入力を行うのか？
* プロジェクトの進捗管理、リソースの管理、予算管理、品質向上のために、工数入力をします。

# 予定工数入力のやり方

## 新規・追加機能開発の場合
* 自身のスキルを考慮し予定工数入力する
* 未知の領域がある場合は、その分をバッファを取る。
* 仕様の理解に時間がかかる場合は、その分の時間も考慮する。

## 既存のバグを修正する場合
予定工数入力する時はチケットの内容と実装を確認し、
* フロントエンドに問題がある
* バックエンドに問題がある
* 両方に問題がある
の３つの内のどのパターンか最初に分類する。

* もし、分類することができなければPMに助けを求める。

* フロントエンドもしくはバックエンドに問題がある場合は、そのまま予定工数入力する。

* 両方に問題がある場合は、フロントエンドとバックエンドそれぞれに対し予定工数入力をする。


## イレギュラーなパターンとして
* フロントエンドの問題だと想定し作業をしていたら、バックエンドの問題だと気づいた場合
* バックエンドの問題だと想定し、作業をしていたらフロントエンドの問題だと気づいた場合
* 実装を進めていて、想定していた時間を大幅にオーバーしそうな場合
上記の場合は、再度その時点で、PMに相談して予定工数入力を再度やり直す。

## その他
* 自分の知らない領域の問題がチケットに含まれる場合
PMと相談して、バッファを取るなどする。

## PMに相談するタイミング
* PMが予定した時間よりも大幅に遅れそうな時
* 予定工数入力後、予定よりも大幅に遅れそうな時
* 予定工数入力を立てられない時
* 実装方法が全くわからない時

## PMに相談する時は
* チャットやメタライフで遠慮せずにまずは話しかける
